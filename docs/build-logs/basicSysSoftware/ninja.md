# Package: Ninja

**Version:** 1.13.2
**Chapter:** 8.58 (System Software)
**SBU Estimate:** 0.2 SBU

## Overview
The Ninja package provides a highly optimized, minimalist build system deliberately designed to focus on speed. Unlike `make`, which is designed to be written by humans, Ninja is designed to have its input files generated by higher-level build systems (like CMake or Meson) and execute them as fast as possible by running builds in parallel.

Key binaries included:
* `ninja` (The core build system executable)

## Security / Custom Modifications
- **Resource Exhaustion Prevention (The `NINJAJOBS` Patch):** Applied a critical C++ source code patch via `sed` to `src/ninja.cc`. By default, Ninja aggressively maximizes parallel processing (CPU cores + 2), which can easily trigger Out-Of-Memory (OOM) kills or thermal throttling on constrained systems, especially when Ninja is invoked silently by a parent build script. This patch forces Ninja to respect a user-defined `NINJAJOBS` environment variable, ensuring the administrator retains strict control over system resource allocation during compilation.

## Build Notes
- **Required Disk Space:** 43 MB
- **Process:** Built natively using the previously installed Python 3 environment (`python3 configure.py --bootstrap --verbose`). The `--bootstrap` flag forces Ninja to rebuild itself for the current architecture.
- **Test Suite:** Explicitly skipped. The formal test suite requires `cmake`, which is not yet installed in the chroot. However, because Ninja successfully builds itself during the bootstrap phase, its core compilation and parallel execution logic are implicitly validated.
- **Installation:** Manual installation using standard `install` commands to place the binary in `/usr/bin/` and configure both `bash` and `zsh` shell completions.
