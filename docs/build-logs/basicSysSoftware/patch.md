# Package: Patch

**Version:** 2.8
**Chapter:** 8.72 (System Software)
**SBU Estimate:** 0.1 SBU

## Overview
The Patch package contains the essential `patch` utility. This program modifies existing files by applying a "patch" file, which is typically a difference listing generated by the `diff` program (installed previously via Diffutils). It is heavily utilized throughout the Linux From Scratch build process (and standard Linux administration) to seamlessly apply bug fixes, security updates, and custom modifications to raw source code prior to compilation.

Key binaries included:
* `patch` (The core file modification engine)

## Security / Custom Modifications
- **System-Wide FHS Compliance:** Configured directly with `--prefix=/usr` to ensure the binary is installed securely into the standard system execution path. Unlike many previous packages, the upstream `patch` source code requires no custom `sed` hacks or LFS-specific vulnerability patches to compile cleanly in the modern environment.

## Build Notes
- **Required Disk Space:** 14 MB
- **Process:** Compiled cleanly in-tree utilizing standard GNU Autotools configurations.
- **Test Suite:** Executed `make check` to ensure the patch parsing logic and file modification algorithms perform accurately without corrupting target files.
- **Installation:** Standard `make install` execution.
